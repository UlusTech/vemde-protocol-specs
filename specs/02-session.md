# 02 — Session

## Overview

A session is a live collaborative context owned by a Vemde server. The session lifecycle begins when the first peer connects and ends when the server terminates it. The protocol does not define when a server must start or stop a session — that is an implementation decision.

## Lifecycle

```
Client                          Server
  |                               |
  |--- vemde.session.hello ------>|
  |<-- vemde.session.welcome -----|
  |                               |
  |     (session active)          |
  |                               |
  |--- vemde.session.leave ------>|
  |<-- vemde.session.left --------|  (broadcast to all peers)
```

When a new peer joins an active session, the server broadcasts `vemde.session.joined` to all existing peers, and sends the new peer a `vemde.session.welcome` with the current peer list.

## Messages

### `vemde.session.hello` — Client → Server

Sent by a client to initiate a session connection.

**Protobuf**
```proto
message SessionHello {
  string peer_id      = 1;  // UUID, generated by client
  string display_name = 2;  // optional, human-readable
  repeated string capabilities = 3;  // list of supported vemde feature strings
}
```

**JSON**
```json
{
  "peerId": "uuid-v4",
  "displayName": "alice",
  "capabilities": []
}
```

---

### `vemde.session.welcome` — Server → Client

Sent to a newly connected client. Contains full current session state.

**Protobuf**
```proto
message SessionWelcome {
  string session_id   = 1;
  repeated Peer peers = 2;
}

message Peer {
  string peer_id      = 1;
  string display_name = 2;
  map<string, string> metadata = 3;  // plugin-contributed identity fields
}
```

**JSON**
```json
{
  "sessionId": "uuid-v4",
  "peers": [
    {
      "peerId": "uuid-v4",
      "displayName": "alice",
      "metadata": {}
    }
  ]
}
```

---

### `vemde.session.joined` — Server → All Clients

Broadcast when a new peer joins.

**Protobuf**
```proto
message SessionJoined {
  Peer peer = 1;
}
```

**JSON**
```json
{
  "peer": {
    "peerId": "uuid-v4",
    "displayName": "bob",
    "metadata": {}
  }
}
```

---

### `vemde.session.leave` — Client → Server

Sent by a client that is intentionally disconnecting.

**Protobuf**
```proto
message SessionLeave {
  string peer_id = 1;
}
```

**JSON**
```json
{
  "peerId": "uuid-v4"
}
```

---

### `vemde.session.left` — Server → All Clients

Broadcast when a peer leaves or disconnects.

**Protobuf**
```proto
message SessionLeft {
  string peer_id = 1;
  string reason  = 2;  // "leave" | "disconnect" | "kicked"
}
```

**JSON**
```json
{
  "peerId": "uuid-v4",
  "reason": "disconnect"
}
```

## Connection Loss

Connection loss behavior is a client implementation decision. The protocol defines only `vemde.session.left` with `reason: "disconnect"` as the server-side signal. What the client does after receiving this (or after losing transport) is not specified.
